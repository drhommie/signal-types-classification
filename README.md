# Signal Types Classification

Сессионный проект по курсу машинного обучения без учителя.

## Цель проекта

Цель работы — выполнить автоматическую кластеризацию сигналов сцинтилляционного детектора.

Необходимо разделить 23 479 сигнала на 3 кластера:

- первый кластер — сигналы первого типа частиц;
- второй кластер — сигналы второго типа частиц;
- третий кластер — аномальные, смешанные или неидентифицируемые сигналы.

Задача решается методами машинного обучения без учителя.

## Что смотреть

Основной файл с финальным решением:

**Final_version.ipynb**

В этом ноутбуке находится итоговое решение проекта:

- исследовательский анализ данных (EDA);
- предобработка сигналов;
- создание признаков;
- анализ и выбор признаков;
- PCA;
- подбор гиперпараметров;
- сравнение моделей кластеризации;
- выбор финальной модели;
- создание submission-файла;
- итоговые выводы.

Дополнительные файлы:

- **Initial version_ML_project.ipynb** — первая версия проекта;
- **Final_version_compare_to_initial.ipynb** — финальная версия с комментариями, где отмечены изменения относительно первой версии.

## Использованные модели

В проекте сравнивались несколько моделей кластеризации:

- KMeans;
- MiniBatchKMeans;
- Gaussian Mixture.

Финальная модель:

- KMeans;
- `n_clusters=3`;
- `random_state=42`;
- `n_init=100`.

## Воспроизводимость

Для воспроизводимости результата в ноутбуке зафиксированы случайные состояния:

- используется `np.random.seed(42)`;
- в моделях указан `random_state=42`;
- для KMeans задан параметр `n_init=100`;
- итоговый submission формируется одним и тем же кодом.

При повторном запуске ноутбука на тех же данных должен получиться тот же результат кластеризации и тот же submission-файл.

## Как запустить проект

1. Скачать репозиторий.
2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Открыть основной ноутбук:

```text
Final_version.ipynb
```

4. Запустить все ячейки сверху вниз.

## Краткий итог

В работе был проведён анализ сигналов, выполнена предобработка, рассчитаны признаки формы сигнала и обучены модели кластеризации.

Для финального решения выбран KMeans, так как он показал лучший результат среди проверенных моделей, даёт воспроизводимый результат и хорошо подходит для интерпретации задачи кластеризации сигналов.
